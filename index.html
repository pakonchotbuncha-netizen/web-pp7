🎯 กรอบการออกแบบระบบประเมินเครดิตใหม่ (จาก Google Sheet)

ส่วนที่ 1: โครงสร้างการประเมิน
1.1 ใช้ BARS เป็นเกณฑ์หลัก
พฤติกรรมที่สังเกตได้จริง
1.2 แยกตามกลุ่มงาน
Core + Functional competency
1.3 ใช้ 5 มุมมอง 360°
น้ำหนัก: หัวหน้า 30%, เพื่อน 25%
ส่วนที่ 2: คำถามและเกณฑ์
2.1
จำนวนคำถาม: 15-20 ข้อ, 4 competency, 8-12 ข้อ/competency
2.2
โครงสร้าง BARS: 5 ระดับ พร้อมพฤติกรรมตัวอย่าง
2.3
แยก 2 ด้าน: ทักษะ (Skills) + ทุนองค์กร (Corporate Capital)
ส่วนที่ 3: กระบวนการดำเนินการ
3.1
Timeline: เตรียม 2 wk → เก็บข้อมูล 2-3 wk → วิเคราะห์ 1-2 wk → แจ้งผล 1 wk → Coaching 2 wk → Follow-up ทุก 2-3 เดือน
3.2
Anonymity & Confidentiality: ไม่รู้ตัวผู้ตอบ, ≥3 rater/มุมมอง
3.3
แยกจากเงินเดือน/โปรโมท: 360 เพื่อพัฒนา, ค่าจ้างใช้ KPI
3.4
Follow-up: Coaching session ภายใน 2 สัปดาห์ + Development plan
ส่วนที่ 4: การออกแบบใน Web PP7
4.1
Entity: EvaluationCycle (รอบปี), Rater Assignment, Scoring, Follow-up
4.2
7 หน้าจอ: ตั้งรอบประเมิน, ตั้ง rater, ทำแบบประเมิน, ดูผล, coaching plan, dashboard, follow-up
4.3
ระบบแจ้งเตือน: แจ้ง rater ที่ยังไม่ตอบ, coaching deadline
4.4
Dashboard: Traffic light, Radar chart (5 มุมมอง), เปรียบเทียบข้ามปี
ส่วนที่ 5: เปรียบเทียบ เก่า vs ใหม่
เกณฑ์ แบบเก่า (ความรู้สึก) แบบใหม่ (BARS + 360)
เกณฑ์ให้คะแนน รู้สึกดี/ไม่ดี ให้คะแนนตามอารมณ์ พฤติกรรมที่สังเกตได้จริง 5 ระดับ พร้อม anchor
มุมมอง หัวหน้าคนเดียว หรือไม่กี่คน 5 มุมมอง (ตนเอง หัวหน้า เพื่อน ลูกน้อง ลูกค้า)
เกณฑ์ เกณฑ์เดียวกันทั้งองค์กร แยกตามกลุ่มงาน + Core + Functional competency
Follow-up ประเมินปีละครั้ง แล้วจบ Coaching + Development plan + Follow-up ทุก 2-3 เดือน
วัตถุประสงค์ ใช้ตัดสินเงินเดือน/โปรโมท เพื่อพัฒนา — ค่าตอบแทนใช้ KPI ต่างหาก
ส่วนที่ 6: แผนดำเนินการ (ให้เสร็จภายในปี 2569)
Phase 1
ออกแบบ (1-2 เดือน)
กำหนด competency, ออกแบบ BARS 5 ระดับ, กำหนดน้ำหนัก, ออกแบบ entity
Phase 2
พัฒนา Web PP7 (2-3 เดือน)
สร้าง 7 หน้าจอ, ระบบ rater, scoring, แจ้งเตือน, dashboard + radar chart
Phase 3
ทดลองใช้ (1 เดือน)
เลือก BU นำร่อง 1 กลุ่ม, รันรอบทดลอง, เก็บ feedback, ปรับปรุงระบบ
Phase 4
ใช้จริงทั้งองค์กร
ฝึกอบรม, เปิดรอบประเมินจริง, Coaching ทุก 2-3 เดือน, วัดผล
