## 🔹 แถวที่ 1 — PP7 Core (กระบวนการหลัก)

### 1. 📌 สรุปผู้บริหาร (Executive Dashboard)

**ความหมาย:**  
Tab แรกที่เปิดขึ้นมา — เป็น **Dashboard ภาพรวมทั้งระบบ PP7** สำหรับผู้บริหารใช้ดูสถานะแบบเรียลไทม์ ตัดสินใจเชิงกลยุทธ์

**สิ่งที่แสดง:**
- **KPI Cards 4 ตัว:** บุคลากรทั้งหมด, Match Rate, Avg Performance Score, Welfare Payout/เดือน — คลิกตัวเลขแต่ละตัวได้เพื่อดูรายละเอียดเจาะลึก (Drill-down)
- **สถานะกระบวนการ P1-P7:** แสดงสถานะแต่ละกระบวนการ (✅ ปกติ / ⚠️ รอประเมิน)
- **กราฟแนวโน้มบุคลากรรายเดือน** และ **Sourcing Channels** (แหล่งรับสมัคร)
- **Audit Summary Table:** ตารางสรุปสถานะเอกสารของแต่ละกระบวนการ (เอกสารครบ, WI/PM ครบ, หมายเหตุ)
- **AI Insights:** กล่องวิเคราะห์อัตโนมัติจาก AI แบ่งเป็น 3 ด้าน — แสวงหา, ประเมิน+จับคู่, ประเมิน+พัฒนา
- **ตารางตัวเลขสำคัญรายฝ่าย:** จำนวน, Avg Score, Turnover, สถานะแต่ละ BU

**การใช้งาน:**
- เปิด Tab นี้เป็น **หน้าแรก** ทุกครั้งเพื่อดูภาพรวม
- คลิก KPI ตัวเลขเพื่อดูรายละเอียดเจาะลึก
- ดู AI Insights เพื่อเห็นประเด็นที่ต้องสนใจทันที
- ใช้ Filter ด้านบน (ฝ่าย, ระดับ) เพื่อกรองข้อมูล
- Export PDF/Excel ได้จาก Toolbar

### 2. 🔎 แสวงหา (Recruit / P1)

**ความหมาย:**  
กระบวนการ **P1 — แสวงหาบุคลากร** ตั้งแต่เปิดรับสมัครจนถึงส่งต่อผู้สมัครเข้าหยั่งประเมิน

**สิ่งที่แสดง:**
- **AI Screening Banner:** แสดงผู้สมัครที่ AI คัดกรองอัตโนมัติ พร้อม % Match และคำแนะนำ
- **KPI 4 ตัว:** ตำแหน่งเปิดรับ, ใน Pipeline, ผ่านสัมภาษณ์, Offer Sent
- **Flow 5 ขั้นตอน:** เปิดรับสมัคร → รับสมัคร → คัดกรอง → สัมภาษณ์ → ส่งต่อ (เข้าหยั่งประเมิน)
- **ตารางผู้สมัคร:** ค้นหาได้, แสดงสถานะแต่ละคน
- **ตาราง Recruitment Pipeline:** ขั้นตอนการรับสมัครตั้งแต่ประกาศจนรับเข้า

**การใช้งาน:**
- ใช้ดูว่า **ตำแหน่งไหนว่าง** มีผู้สมัครกี่คน
- ดู AI Screening เพื่อเห็นผู้สมัครที่น่าสัมภาษณ์
- ติดตาม Pipeline ว่าผู้สมัครอยู่ขั้นตอนไหน
- เมื่อสัมภาษณ์เสร็จ → ส่งต่อไปยัง Tab "หยั่งประเมิน"

### 3. 🧭 หยั่งประเมิน (Assess / P2)

**ความหมาย:**  
กระบวนการ **P2 — หยั่งประเมินสมรรถนะ** วัด Core Competency (CC) ของผู้สมัครก่อนรับเข้า

**สิ่งที่แสดง:**
- **KPI 5 ตัว:** Skill Gate ผ่าน, Attitude Fit, Behavior Ready, พร้อมเข้า Matching, รอประเมินเพิ่ม
- **โครงสร้างการประเมิน 4 มิติ:**
  1. **Skill Simulation Gate** — จำลองสถานการณ์ทำงานจริง (ต้องผ่านก่อน)
  2. **Attitude/Culture Fit** — วัดทัศนคติจากปรัชญาองค์กร PKG
  3. **Behavioral Scenario** — สถานการณ์จำลองดูวิธีตัดสินใจ
  4. **Recommendation Engine** — วิเคราะห์ความเหมาะสมเทียบตำแหน่งที่เปิดรับ
- **ตารางคะแนน:** Skill Score, Attitude Score, Behavior Score, Position Fit Score
- **ตารางผู้สมัครในกระบวนการ:** แสดงผลแต่ละคน (ผ่าน/รอ/ไม่ผ่าน)
- **ตัวอย่างชุดประเมิน:** คำถามทัศนคติจากปรัชญาองค์กร, โจทย์จำลองสถานการณ์ตามสายงาน
- **AI Bias Detection + Trend Analysis:** ตรวจสอบอคติผู้ประเมิน, วิเคราะห์แนวโน้มคะแนน

**การใช้งาน:**
- ใช้เมื่อผู้สมัครมาจาก Tab "แสวงหา" แล้ว
- ประเมิน 4 มิติ → ถ้าผ่าน → ส่งต่อไป Tab "จับคู่คนกับงาน"
- ดู AI Bias Detection เพื่อตรวจสอบความเที่ยงตรงของการประเมิน
- ใช้ตัวอย่างชุดประเมินเป็นแนวทางสร้างแบบประเมินจริง

### 4. 🤝 จับคู่คนกับงาน (Match / P3)

**ความหมาย:**  
กระบวนการ **P3 — จับคู่คนกับงาน** นำผู้ที่ผ่านการประเมิน (P2) มาจับคู่กับตำแหน่งที่เหมาะสม

**สิ่งที่แสดง:**
- **KPI 4 ตัว:** Match Rate, จับคู่สำเร็จ Q1, Evaluation Ready, Avg Match Score
- **Flow 3 ขั้นตอน:** Recruitment/Assessment → Matching → Evaluation → Development
- **AI Matching Recommendations:** แนะนำจับคู่ Top 3 พร้อม % Match
- **ตารางรายการจับคู่ล่าสุด:** แสดง Match Score, ตำแหน่ง, Evaluation, สถานะ
- **ตาราง JD, LIA & Competency Fit:** แสดง Job Description, LIA, Competency Fit, Gap, Recommendation, Approval Flow, Transfer Fit

**การใช้งาน:**
- ใช้เมื่อผู้สมัครผ่าน "หยั่งประเมิน" แล้ว
- ดู AI Recommendations เพื่อเห็นการจับคู่ที่ดีที่สุด
- ตรวจสอบ Competency Fit ว่าคนตรงกับตำแหน่งไหม
- ถ้ามี Gap → ระบุแผนพัฒนาก่อนอนุมัติ
- เมื่อจับคู่สำเร็จ → ส่งต่อไปยัง Tab "ประเมินผล"

### 5. 📊 ประเมินผล (Performance / P4)

**ความหมาย:**  
กระบวนการ **P4 — ประเมินผลงาน** วัดผล Core Competency หลังลงงานจริง แบ่งเป็น 2 ประเภทหลัก:
- **ประเมินหลังทำงานปกติ** (Performance Evaluation)
- **ประเมินเครดิตประจำปี** (Annual Credit Evaluation — 360 องศา)

**สิ่งที่แสดง:**
- **KPI 5 ตัว:** สมาชิกเข้าประเมิน, ผ่านจาก Matching แล้ว, ประเภทการประเมิน, ต้องพัฒนา, พร้อมยืนยันบทบาท
- **โครงระบบประเมินผลที่เชื่อมจาก Matching:** Entry → Evaluation Type → Evaluator Workflow → Output
- **ประเภทการประเมิน:** ทดลองงาน, สัญญาจ้าง, โอนย้าย, เลื่อนตำแหน่ง, ประเมินประจำรอบ
- **AI Performance Insights:** Trend Prediction, Development Recommendations, Promotion Candidates
- **กราฟคะแนนรายไตรมาส** และ **Performance Distribution**
- **Annual Credit Evaluation:** โครงคะแนนเครดิต 4 Model (Community Member, Servant/Advisor, PKG Service Team, AAMG Performance)
- **พื้นที่ออกแบบประเมิน 360 องศา:** โจทย์ออกแบบ, แนวทางวิจัย, องค์ประกอบข้อมูล, งานที่ต้องทำต่อ
- **ตารางคำถามประเมินเครดิต ปี 69:** แบ่งหมวดตาม มาตรฐาน 9, อุปนิสัย 8, ค่านิยม 6 รับ, From PVP
- **ตารางผลการประเมิน:** แสดงผลรายบุคคล พร้อม Link ไป Development/Compensation

**การใช้งาน:**
- ใช้ประเมินพนักงานที่ผ่าน "จับคู่คนกับงาน" แล้ว
- เลือกประเภทการประเมินตามสถานะ (ทดลองงาน/ประจำ/โอนย้าย/เลื่อนตำแหน่ง)
- ดู AI Promotion Candidates เพื่อเห็นคนที่พร้อมเลื่อนตำแหน่ง
- ใช้พื้นที่ 360 องศา ออกแบบการประเมินเครดิตประจำปี
- ผลประเมิน → ส่งต่อไปยัง "พัฒนา", "ค่าตอบแทน", "คุณภาพชีวิต"

### 6. 🎓 พัฒนา (Develop / P5)

**ความหมาย:**  
กระบวนการ **P5 — พัฒนาบุคลากร** ตาม Core Competency ที่บกพร่องจากผลประเมิน (P4)

**สิ่งที่แสดง:**
- **KPI 4 ตัว:** Training Completion, คอร์สเปิด Q1, Avg Score, Budget ใช้ไป
- **Skill Gap Analysis:** ตารางแสดงทักษะที่ Required vs Current → หา Gap → จัดลำดับความสำคัญ
- **AI Development Plan:** ออกแผนพัฒนารายบุคคลอัตโนมัติ พร้อม Training ROI Prediction
- **กราฟ Training Hours by Department**
- **ตารางคอร์สและการพัฒนา:** รายบุคคล (ID, ชื่อ, คอร์ส, ชั่วโมง, คะแนน, สถานะ, วันที่เสร็จ)
- **ตาราง IDP & Progress:** Individual Development Plan, Mentor, Progress %, Learning History, Competency Gap

**การใช้งาน:**
- ใช้เมื่อมีผลประเมิน (P4) แล้ว เห็นว่า CC ไหนต้องพัฒนา
- ดู Skill Gap Analysis เพื่อเห็นว่าทักษะไหนขาดที่สุด
- ใช้ AI Development Plan เพื่อสร้างแผนพัฒนารายบุคคล
- ติดตาม Progress ของแต่ละคนในตาราง IDP
- บันทึกผลการเรียนคอร์สในตารางคอร์ส

### 7. 💰 ค่าตอบแทน (Compensation / P6)

**ความหมาย:**  
กระบวนการ **P6 — ค่าตอบแทนและแรงจูงใจ** ตอบแทนสมาชิกตาม CC และผลงาน

**สิ่งที่แสดง:**
- **KPI 5 ตัว:** เงินเดือนรวม/เดือน, Incentive Payout, Point สะสมรวม, Annual Credit Output, Avg Salary Growth
- **Reward Logic 3 ส่วน:**
  1. **เงินเดือน** — ฐานตามโครงสร้าง + ปรับตาม CC level และ Performance Score
  2. **Incentive** — จ่ายตามผลงานรายเดือน/ไตรมาส = Base × Performance Multiplier × CC Bonus
  3. **Point System** — สะสมจากผลงานพิเศษ, การพัฒนา CC, Mentor, กิจกรรม → แลกรางวัล
- **Annual Credit Output Table:** ผลประเมินเครดิตประจำปี → เกรด (Top/Solid/Develop/Needs Work) → ปรับเงินเดือน + Incentive
- **Compensation Summary by BU:** แยกตาม PKG-1, PKG-2, PKG-3, สำนักงาน
- **Point System Detail:** วิธีได้ Point (ผลงานพิเศษ +50, พัฒนา CC +20, Mentor +30, กิจกรรม +10) + วิธีแลก
- **AI Compensation Insights:** Pay Equity, Budget Forecast, Retention Risk
- **ตารางข้อมูลรายบุคคล:** เงินเดือน, Incentive, Point, Annual Credit, รวมค่าตอบแทน, สถานะ

**การใช้งาน:**
- ใช้คำนวณค่าตอบแทนจากผลประเมิน (P4) และเครดิตประจำปี
- ดู Annual Credit Output เพื่อกำหนดเกรดและปรับเงินเดือน
- ตรวจสอบ Pay Equity ว่ามีความเท่าเทียมไหม
- ใช้ Point System สร้างแรงจูงใจเพิ่มเติม
- ดู Retention Risk เพื่อป้องกันพนักงานลาออก

### 8. 🌿 คุณภาพชีวิต (Quality of Life / P7)

**ความหมาย:**  
กระบวนการ **P7 — คุณภาพชีวิต** ดูแลสมาชิก 5 องค์ประกอบ: สุขภาพ, ความมั่งคั่ง, บรรยากาศ, เวลา, อารมณ์ขัน

**สิ่งที่แสดง:**
- **5 Dimensions Cards:** สุขภาพ (ตรวจสุขภาพ, ประกัน), ความมั่งคั่ง (สหกรณ์, ออมหุ้น), บรรยากาศ (CEW, จับถูก), เวลา (ยืดหยุ่น, WLB), อารมณ์ขัน (Team Building, กิจกรรมสนุก)
- **ตาราง Service Activity:** กิจกรรมแต่ละด้าน, ผู้เข้าร่วม, ความพึงพอใจ, สถานะ
- **Wellbeing Status:** ภาพรวมคุณภาพชีวิต 5 ด้าน
- **AI Wellbeing Analysis:** วิเคราะห์จาก Survey, ลาป่วย, OT, Turnover, Engagement Score
- **ตารางผลสำรวจรายบุคคล:** Engagement, WLB, Stress, Wellbeing Score, สถานะ (ดีเยี่ยม/ควรดูแล/ต้องติดตาม)

**การใช้งาน:**
- ใช้ติดตามคุณภาพชีวิตพนักงานรายบุคคลและรายองค์กร
- ดู Wellbeing Score เพื่อเห็นพนักงานที่ "ต้องติดตาม" (สีแดง)
- วางแผนกิจกรรม Quality of Life จากข้อมูล Service Activity
- ใช้ AI Analysis เพื่อเห็นประเด็นที่ต้องปรับปรุง (เช่น Work-Life Balance ต่ำ)

## 🔹 แถวที่ 2 — Reference & System (ข้อมูลอ้างอิงและระบบ)

### 9. ⚖️ กฎหมายแรงงาน (Labor Law)

**ความหมาย:**  
รวบรวม **กฎหมายคุ้มครองแรงงาน** จากแหล่งภาครัฐ 3 ประเทศ (ไทย, สปป.ลาว, กัมพูชา) แยกตามหมวดใช้งาน HR

**สิ่งที่แสดง:**
- **KPI 4 ตัว:** ช่วงเวลาทบทวน (5 ปี), ประเทศ (3), สถานะ (รอบแรก), แหล่งอ้างอิง
- **ข้อมูลแยกตามประเทศ:**
  - **THAI (ไทย):** เวลาทำงาน, เวลาพัก, วันหยุด, วันลา, งานอันตราย
  - **RPLC (สปป.ลาว):** โครงหมวดพร้อมเติม
  - **RAFCO (กัมพูชา):** โครงหมวดพร้อมเติม
- **ตารางหมวดกฎหมาย:** 16 หมวด (เวลาทำงาน, วันหยุด, วันลา, ค่าจ้าง, ค่าล่วงเวลา, ค่าชดเชย, ฯลฯ)
- **ตารางสถานะข้อมูล:** แสดงว่าหมวดไหนใส่แล้วรอบแรก, มีข้อมูลตั้งต้น, รอเติม

**การใช้งาน:**
- ใช้อ้างอิงกฎหมายแรงงานเมื่อออกแบบนโยบาย HR
- ตรวจสอบว่าการปฏิบัติขององค์กรสอดคล้องกฎหมายไหม
- ใช้ข้อมูลเวลาทำงาน/วันลา เมื่อออกแบบระบบ Attendance

### 10. 📘 ระเบียบ PKG (PKG Rules)

**ความหมาย:**  
**ระเบียบข้อบังคับภายในกลุ่มบริษัท PKG** แยกตามประเทศ (THAI, RPLC, RAFCO)

**สิ่งที่แสดง:**
- **ข้อมูลแยกตามประเทศ 3 ประเทศ:**
  - **THAI (ไทย):** กติกามารยาท, เวลาทำงาน, การลงเวลา (ใส่แล้วรอบแรก)
  - **RPLC (สปป.ลาว):** โครงหมวดพร้อมเติม
  - **RAFCO (กัมพูชา):** โครงหมวดพร้อมเติม
- **รายละเอียดรายหมวด THAI:** กติกามารยาท, เวลาทำงาน/พัก, การลงเวลา, โครงหัวข้อที่จะเติมต่อ
- **ตารางสถานะข้อมูล:** แสดงว่าหมวดไหนใส่แล้ว/มีข้อมูลตั้งต้น/รอเติม
- **ตาราง Policy, WI, PM & ระเบียบภายใน:** รหัสเอกสาร, ชื่อ, สถานะ, ปรับปรุงล่าสุด
- **ตารางวินัย:** หมวดวินัย, การกระทำผิด, บทลงโทษ, ระดับโทษ

**การใช้งาน:**
- ใช้อ้างอิงระเบียบภายในเมื่อจัดการเรื่องวินัย
- ตรวจสอบการลงเวลา (08:15 น.) เมื่อทำ Attendance
- ใช้กฎมารยาทเป็นฐานประเมิน Attitude/Behavior
- เติมข้อมูลประเทศลาว/กัมพูชาเมื่อพร้อม

### 11. 🔄 Import/Export (Data Exchange)

**ความหมาย:**  
**ศูนย์กลางการนำเข้าและส่งออกข้อมูล** สำหรับ HR, Operation และผู้บริหาร

**สิ่งที่แสดง:**
- **KPI 4 ตัว:** Import Modules (5), Export Formats (3), Validation (Preview), Audit Log (Enabled)
- **Import Center:** 5 Module — Recruitment Update, Assessment Result, Placement Update, Operation Feed, Master Data
- **Export Center:** 5 ประเภท — Executive Report, Operation Report, Assessment Export, Placement Export, Compensation Summary
- **Workflow มาตรฐาน Import:** ดาวน์โหลด template → กรอก → อัปโหลด → Validation → Preview → Confirm → Log
- **Validation ที่มี:** ตรวจชื่อคอลัมน์, ชนิดข้อมูล, field บังคับ, key อ้างอิง, แจ้ง error รายแถว, สรุปผล
- **สิทธิ์การใช้งาน:** Admin, HR, Operation, Executive — ขอบเขตต่างกัน
- **Target, Field Mapping & Integration Points:** Source → Target → Template → Field Mapping → Validation → Integration Point

**การใช้งาน:**
- ใช้เมื่อต้องการ **นำเข้าข้อมูล** จาก Excel/CSV เข้าสู่ระบบ
- ใช้เมื่อต้องการ **ส่งออกข้อมูล** เป็น PDF/Excel/CSV
- ดาวน์โหลด Template จาก Import Center → กรอก → อัปโหลด → Preview → Confirm
- ดู Field Mapping เพื่อเข้าใจว่าข้อมูลจากไหนไปไหน
- ตรวจสอบสิทธิ์ว่า Role ของตัวเองทำอะไรได้บ้าง
