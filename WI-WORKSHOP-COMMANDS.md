## 📐 7 ขั้นตอนมาตรฐาน (ห้ามข้าม)

### ขั้นตอนที่ 1: อ่านเงื่อนไขก่อนทำ ⚠️ **สำคัญที่สุด**
- เปิดแท็บ `เงื่อนไขปฏิบัติของ OpenClaw` ในชีส
- อ่านกฎทั้งหมด — ห้ามใช้ memory จากครั้งก่อน
- ยืนยันว่าเข้าใจเงื่อนไขปัจจุบัน

### ขั้นตอนที่ 2: ตรวจสอบ C2 → เปลี่ยนสถานะทันที
- ไปที่แท็บ C2 ที่เกี่ยวข้อง (สรรหาและประเมินผล / บริหารหลักสูตร / สวัสดิการและค่าตอบแทน)
- หาแถวของ WI-XXX
- ตรวจสอบ HTTP status ของลิงก์: 200/302/301 = ✅ เข้าถึงได้, 401 = ❌ ต้อง Login, 404 = ❌ ไม่พบ
- **ถ้าลิงก์เข้าถึงได้ → เปลี่ยน ❌ → ✅ ในคอลัมน์สถานะของ C2 ทันที (ก่อนทำ S1-S7!)**
- เปลี่ยนเฉพาะในเซลล์เดิม — ห้ามสร้างเซลล์ใหม่
- **นี่คือขั้นตอนแรกสุด ก่อนดาวน์โหลด drawio หรือเขียน S1-S7**

### ขั้นตอนที่ 3: ดึงเนื้อหาเอกสาร
- ดาวน์โหลดไฟล์ drawio จากลิงก์
- อ่านเนื้อหาทั้งหมด — **ห้ามเดา ห้ามสมมติ**
- ถ้าเปิดไม่ได้ → แจ้งผู้ใช้ ขอลิงก์ใหม่

### ขั้นตอนที่ 4: ทวนว่าจะทำอะไร
- สรุปสั้น ๆ ว่าจะทำ S1-S7 อะไร (ไม่ต้องร่าง全部内容)
- เช่น "จะดึงเนื้อหา WI-XXX มาเขียน S1-S7 ใน P1-4 แถว YY-ZZ, S3=รอ"
- ถ้าผู้ใช้ไม่ทัก → ทำต่อได้เลย

### ขั้นตอนที่ 5: เขียนลงชีส P1-4
- หาแถวว่างหรือแถวที่เคลียร์แล้ว
- เขียน header row + 7 แถว (S1-S7)
- **โครงสร้างแต่ละแถว:**
  - A: `S1` | `S2` | `S3` | `S4` | `S5` | `S6` | `S7`
  - B: หัวข้อขั้นตอน
  - C: เนื้อหา (numbered list)
  - D: ผู้รับผิดชอบ
  - E: สถานะ (`✅` หรือ `รอ`)
- **เขียนเสร็จแล้ว แจ้งผู้ใช้ให้ตรวจสอบ**

### ขั้นตอนที่ 6: ตรวจสอบความถูกต้อง
- เช็ค header มีชื่อกระบวนการ + รหัสเอกสาร
- เช็ค S1-S7 ครบ 7 แถว
- เช็ค A (S labels) ไม่ว่าง
- เช็ค B (หัวข้อ) ถูกต้อง
- เช็ค C (เนื้อหา) มีข้อมูล
- เช็ค D (ผู้รับผิดชอบ) ครบทุกแถว
- เช็ค E (สถานะ) S3 = "รอ" ถ้าผู้ปฏิบัติงานยังไม่กรอก
- ถ้าผิด → แก้ไขทันที

### ขั้นตอนที่ 7: ตรวจสอบ C2
- ยืนยันว่า C2 ถูกเปลี่ยนเป็น ✅ แล้วในขั้นตอนที่ 2 (ก่อนทำ S1-S7)
